## 4. Tech Stack — Stability-First Bootstrap Stack

> **Mandate:** Open-source, self-hosted, stable in production, designed to evolve from internal tool → SaaS.

### 4.1 Core Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                  Next.js 15 Frontend                         │
│    (Internal Dashboard → Client Portal → Self-serve SaaS)   │
├─────────────────────────────────────────────────────────────┤
│              FastAPI (Python) — API Gateway                   │
├──────────┬──────────┬──────────┬────────────┬───────────────┤
│ Discovery│ Outreach │ Content  │ Negotiation│ Reporting     │
│ Agent    │ Agent    │ QC Agent │ Agent      │ Agent         │
├──────────┴──────────┴──────────┴────────────┴───────────────┤
│          LangGraph Orchestrator (State Machine)              │
│          Human-in-the-Loop Interrupt Gates                   │
├─────────────────────────────────────────────────────────────┤
│  PostgreSQL │ Redis │ Qdrant (Vector DB) │ MinIO (Media)    │
├─────────────────────────────────────────────────────────────┤
│  Evolution API (WhatsApp) │ SMTP (Email) │ IG API (DMs)    │
└─────────────────────────────────────────────────────────────┘
Hosted on: Coolify → Hetzner VPS (₹3K/mo → ₹30K/mo at scale)
```

### 4.2 Technology Decisions

| Layer | Technology | Why This? | Alternatives Rejected |
|---|---|---|---|
| **Agent Framework** | **LangGraph (Python)** | Stateful agent graphs, human-in-the-loop interrupt gates, battle-tested at scale, LangSmith observability built-in | CrewAI (too high-level, less control), AutoGen (MS dependency) |
| **Backend API** | **FastAPI + Celery** | Python-native (aligns with LangGraph), async-first, Celery for background campaign jobs | Django (too heavy), Flask (no async), Node (Python ecosystem mismatch) |
| **Frontend** | **Next.js 15 (App Router + TypeScript)** | SSR for dashboards, great DX, React ecosystem, evolves from internal tool to SaaS portal | Svelte (smaller ecosystem), Vue (less enterprise adoption) |
| **Database** | **PostgreSQL 16 (self-hosted) + Drizzle ORM** | Relational data (campaigns, invoices, influencers), JSONB for flexible fields, most battle-tested DB, open-source | MySQL (less feature-rich), MongoDB (wrong for relational campaign data) |
| **Vector DB** | **Qdrant (self-hosted)** | Influencer profile embeddings for semantic search ("find food bloggers who do casual reviews"), Rust-based = fast, simple API | Pinecone (cloud-only, expensive), Weaviate (heavier), pgvector (OK for small scale) |
| **Workflow Engine** | **Temporal (self-hosted)** | Campaigns run for days/weeks — Temporal guarantees delivery, handles retries, supports long-running workflows | n8n (good for glue, not for stateful workflows), Inngest (cloud-only) |
| **WhatsApp** | **Evolution API (self-hosted)** | Open-source WhatsApp connector, free, self-hosted, supports multi-device | Twilio (expensive per message), WATI (SaaS dependency) |
| **Email** | **Postal (self-hosted SMTP)** | Open-source, self-hosted mail server, handles bulk outreach without per-email costs | SendGrid (per-email pricing), SES (AWS dependency) |
| **LLM (Manager)** | **Claude 3.5 Sonnet (Anthropic)** | Best at nuanced text generation (outreach messages, negotiation, content review), strongest reasoning | GPT-4 (good but pricier), Gemini (less nuanced for marketing copy) |
| **LLM (Worker tasks)** | **Llama 3 70B via Ollama (self-hosted)** | Free for repetitive tasks (data enrichment, scoring, categorization), keeps costs <₹5K/mo at scale | Mixtral (good alternative), GPT-3.5 (pay-per-call adds up) |
| **Scraping** | **Firecrawl (web) + Social Media APIs** | Firecrawl is LLM-optimized web scraping, social APIs for structured profile data | Apify (expensive at scale), Bright Data (overkill for early stage) |
| **CRM** | **Odoo (self-hosted)** | Free, open-source, ERP-grade CRM, eventually handles invoicing + HR + inventory | HubSpot (SaaS, expensive at scale), Freshsales (SaaS dependency) |
| **Payments** | **Razorpay (India)** | Standard Indian payment gateway, handles invoicing + commission payouts | Stripe (not ideal for India-domestic), PayU (less developer-friendly) |
| **File Storage** | **MinIO (self-hosted)** | S3-compatible, stores video drafts, contracts, reports — free, self-hosted | AWS S3 (cloud dependency), Cloudflare R2 (good alternative) |
| **Observability** | **LangSmith + Grafana + Loki** | LangSmith traces every agent decision for debugging; Grafana for infra metrics; Loki for logs | Arize Phoenix (good but less integrated), Datadog (expensive) |
| **Hosting** | **Coolify on Hetzner VPS** | Open-source PaaS (like self-hosted Vercel), Hetzner offers best price/performance globally, ~€12/mo to start | Railway (SaaS, costs scale fast), Vercel (violates self-hosted mandate) |

## 5. Marketing Psychology Playbook for Growth

### 5.1 Acquiring Brands (Restaurants)

| Principle | How to Apply |
|---|---|
| **Reciprocity** | Offer one free barter collaboration to the restaurant. Once they see results, they feel obligated to continue — now as paying clients. |
| **Social Proof / Bandwagon** | "We work with 15 restaurants in South Delhi" — show logos, video results, engagement numbers. Delhi restaurant owners follow each other closely. |
| **Loss Aversion** | "Your competitor XYZ just got 50K views from our influencer campaign. You're leaving footfall on the table." Frame as what they're *losing*, not gaining. |
| **Anchoring** | Show a competitor's agency quote first (₹50K-₹2L for a campaign), then show your price (₹15K). The contrast makes your pricing feel like a steal. |
| **Scarcity** | "We only onboard 5 new restaurants per month in your area" — creates urgency and exclusivity. |
| **Authority** | Partner with 1-2 well-known food bloggers early. Their endorsement of your agency carries weight with restaurant owners. |
| **IKEA Effect** | Let the restaurant owner pick influencers from a shortlist. Their involvement in selection increases perceived campaign value. |
| **Foot-in-the-Door** | Start with a ₹5K barter-only deal. Once trust is built, upsell to paid collaborations + website + ads (₹15-25K total). |

### 5.2 Acquiring Influencers

| Principle | How to Apply |
|---|---|
| **Zero-Price Effect** | "Join our platform for free. We bring you paid brand deals." The word "free" creates irrational preference over competitors who charge onboarding fees. |
| **Endowment Effect** | Once an influencer completes 1 deal through you, they feel ownership of the relationship. They're psychologically reluctant to leave for a competitor. |
| **Goal-Gradient Effect** | Show progress: "You're 2 deals away from Silver status (higher commission, priority brand access)." Gamify the relationship. |
| **Commitment & Consistency** | Get a small "yes" first — "Would you be open to a free meal at a restaurant in exchange for a reel?" Then escalate to paid deals. |
| **Social Proof** | "127 food creators in Delhi are already getting deals through us." Numbers create FOMO (Fear of Missing Out). |
| **Unity Principle** | Use language that creates belonging: "Join the VibMedia Creator Community." Make them feel part of a tribe, not a transaction. |
